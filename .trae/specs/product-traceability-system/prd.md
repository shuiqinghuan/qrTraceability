# 农产品溯源系统 - 产品需求文档

## 1. 产品概述

农产品溯源系统是一个展示农产品全生命周期信息的Web应用，通过直观的视觉呈现让消费者了解产品的品种信息、生长环境和采收质量，增强产品可信度和品牌价值。

**目标用户：**
- 终端消费者：希望了解农产品来源和质量信息
- 农产品企业：展示产品溯源信息，提升品牌信任度
- 质检机构：查阅产品采收质量数据

**核心价值：**
- 通过可视化的信息展示，让消费者"看得见"农产品的品质
- 建立从田间到餐桌的信任桥梁
- 提升农产品的品牌溢价能力

## 2. 核心功能模块

### 2.1 产品基础信息模块

展示产品的品种和种植信息，以"枣甜5号"为例：

**信息字段：**
- 品种名称：枣甜5号
- 品种编码：4395
- 定植地点：XX省XX市XX农场
- 定植时间：YYYY-MM-DD
- 种植批次号：可选字段

**展示特点：**
- 品种编码采用标签式设计，突出显示
- 地点信息配合地图图标，增强可读性
- 时间信息清晰展示，格式统一

### 2.2 产品多媒体展示模块

以图片和视频为核心展示内容：

**图片展示：**
- 产品实物照片
- 种植基地环境照片
- 采收现场照片
- 包装照片

**视频展示：**
- 产品介绍视频
- 种植过程短视频
- 采收过程视频

**展示特点：**
- 视频和图片占据页面主体位置（60%+视觉权重）
- 图片采用网格布局或轮播展示
- 视频支持自动播放/暂停控制
- 响应式设计，移动端友好

### 2.3 采收质量信息模块

详细展示产品的采收和品质数据：

**质量指标：**
| 指标 | 说明 | 数据类型 |
|------|------|----------|
| 采收起始时间 | 开始采收的日期时间 | YYYY-MM-DD |
| 采收终止时间 | 结束采收的日期时间 | YYYY-MM-DD |
| 糖度 | 水果甜度指标 | Brix值（%） |
| 重量 | 单果重量或总重量 | 克（g）/ 千克（kg） |
| 口感描述 | 产品口感特点 | 文本描述 |
| 适应人群 | 适宜人群范围 | 多选标签 |
| 品质小结 | 综合品质评价 | 文本描述 |

**展示特点：**
- 数据采用卡片式布局，分组清晰
- 糖度和重量可配合图标或进度条可视化
- 适应人群采用标签云展示
- 品质小结使用引用样式突出显示

## 3. 用户界面设计

### 3.1 设计风格定位

**整体风格：** 简约现代 · 自然清新

**设计理念：**
- 以产品（图片/视频）为核心，文字信息为辅助
- 大量留白，突出产品本身
- 自然的配色方案，呼应农产品特性
- 柔和的圆角设计，传递亲和力

### 3.2 视觉设计规范

**色彩方案：**
```
主色调（Primary）：#4A7C59（森林绿）
- 象征自然、健康、有机
- 用于：主按钮、重要标题、强调元素

辅助色（Secondary）：#F5E6D3（暖米色）
- 营造温馨、自然的氛围
- 用于：背景色、卡片底色

强调色（Accent）：#E8A838（丰收金）
- 代表丰收、品质、活力
- 用于：图标、重要数据、徽章

中性色（Neutral）：#2D3436（深灰）
- 用于：正文文字、次要信息

背景色（Background）：#FAFAFA（浅灰白）
- 干净的背景，突出内容
```

**字体选择：**
```
标题字体：Noto Serif SC（思源宋体）
- 宋体风格，传达传统与品质感
- 用于：大标题、章节标题

正文字体：Noto Sans SC（思源黑体）
- 现代简洁，易于阅读
- 用于：正文、说明文字、数据标签
```

**圆角与阴影：**
```
圆角半径：
- 小圆角：8px（标签、小按钮）
- 中圆角：16px（卡片、输入框）
- 大圆角：24px（模态框、大容器）

阴影层次：
- 轻度阴影：0 2px 8px rgba(0,0,0,0.08)
- 中度阴影：0 4px 16px rgba(0,0,0,0.12)
- 重度阴影：0 8px 32px rgba(0,0,0,0.16)
```

**动效设计：**
```
页面加载：渐入效果，opacity 0→1，duration 400ms
图片悬停：轻微放大 scale(1.02)，duration 300ms
按钮交互：背景色渐变，duration 200ms
滚动视差：图片轻微位移，增强层次感
```

- 质量数据卡片2列网格

### 3.4 组件设计细节

**信息卡片组件：**
- 白色背景，16px圆角，12px内边距
- 中度阴影
- 顶部可添加彩色条带标识类别
- Hover时轻微上浮效果

**标签组件（Badge）：**
- 圆角胶囊形状（border-radius: 9999px）
- 浅色背景 + 深色文字
- 可选：带图标或不带图标

**媒体展示组件：**
- 图片：圆角16px，支持点击放大预览
- 视频：圆角16px，播放控件半透明
- 支持懒加载，优化性能

**数据展示卡片：**
- 大字号数字展示核心指标
- 小字号标签说明指标含义
- 可配合简单图表或进度条

**品质小结组件：**
- 引用样式，左侧彩色竖线
- 较大字号，舒适行高
- 可配合头像或图标增强信任感

## 4. 响应式设计

**断点设置：**
```
桌面端：≥1024px（默认设计）
平板端：768px - 1023px（两列布局）
移动端：<768px（单列布局）
```

**适配策略：**
- 桌面优先设计
- 移动端保持核心功能完整
- 图片和视频始终保持清晰
- 触摸友好的交互区域（最小44px）

## 5. 技术选型（前端）

**框架选择：**
- Vue 3 + Composition API
- Vite（构建工具）
- Vue Router（路由管理）
- Pinia（状态管理，可选）

**UI库建议：**
- Tailwind CSS（推荐，灵活定制）
- 或 Element Plus（组件丰富）

**多媒体处理：**
- 图片：支持懒加载、WebP格式
- 视频：支持HLS流媒体、多种格式
- 预览：图片灯箱组件

**性能优化：**
- 路由懒加载
- 图片懒加载
- 组件按需引入
- CSS动画优先（GPU加速）

## 6. 页面路由规划

| 路由 | 页面名称 | 功能描述 |
|------|----------|----------|
| `/` | 首页 | 展示溯源产品详情 |
| `/product/:id` | 产品详情页 | 动态加载不同产品信息 |

## 7. 开发优先级

**第一阶段（MVP）：**
1. 单产品详情页面开发
2. 基础信息展示
3. 图片视频展示
4. 采收质量信息展示
5. 响应式适配

**第二阶段（增强）：**
1. 多产品切换
2. 图片灯箱功能
3. 视频播放器优化
4. 数据可视化增强

## 8. 用户体验优化

**加载体验：**
- 骨架屏加载效果
- 图片渐进式加载
- 视频加载指示器

**交互体验：**
- 图片点击放大
- 视频播放控制
- 平滑滚动效果

**信任增强：**
- 明确的品质认证标识
- 可追溯的时间线
- 透明的数据展示
