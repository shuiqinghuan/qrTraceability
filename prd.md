# 农产品溯源系统 PRD 产品需求文档

## 1. 产品概述

### 1.1 产品背景
农产品溯源系统是一个面向消费者的产品信息追溯平台，通过展示农产品的品种信息、种植过程、采收质量等全流程数据，提升产品透明度和消费者信任度。

### 1.2 目标用户
- 农产品消费者
- 农产品生产企业
- 农产品质量监管部门

### 1.3 技术栈
- 前端：Vue 3 + Vite
- 后端：Python Django
- 数据库：SQLite

## 2. 功能需求

### 2.1 产品基本信息模块

#### 2.1.1 功能描述
展示农产品的基础品种信息，包括品种名称、品种编码、定植地点、定植时间。

#### 2.1.2 数据字段
| 字段名称 | 字段说明 | 数据类型 | 示例值 |
|---------|---------|---------|--------|
| 品种名称 | 产品的品种名称 | String | 枣甜5号 |
| 品种编码 | 产品的唯一编码 | String | 4395 |
| 定植地点 | 产品种植的具体位置 | String | 山东省济南市历城区农业示范园 |
| 定植时间 | 产品开始种植的日期 | Date | 2024-03-15 |

#### 2.1.3 交互说明
- 信息以卡片形式展示
- 鼠标悬停时显示详细信息提示

### 2.2 产品多媒体信息模块

#### 2.2.1 功能描述
展示产品的图片和视频信息，作为页面的主体展示区域。

#### 2.2.2 数据字段
| 字段名称 | 字段说明 | 数据类型 | 示例值 |
|---------|---------|---------|--------|
| 图片列表 | 产品的多张图片 | Array[String] | ["image1.jpg", "image2.jpg"] |
| 视频列表 | 产品的多个视频 | Array[String] | ["video1.mp4"] |
| 媒体类型 | 图片或视频 | Enum | image/video |

#### 2.2.3 交互说明
- 图片支持轮播展示，自动播放间隔5秒
- 视频支持播放、暂停、全屏功能
- 图片支持点击放大预览
- 媒体切换采用平滑过渡动画

### 2.3 产品采收质量信息模块

#### 2.3.1 功能描述
展示产品的采收过程和质量检测信息。

#### 2.3.2 数据字段
| 字段名称 | 字段说明 | 数据类型 | 示例值 |
|---------|---------|---------|--------|
| 采收起始时间 | 开始采收的日期 | Date | 2024-07-01 |
| 采收终止时间 | 结束采收的日期 | Date | 2024-07-15 |
| 糖度 | 产品的糖度值（Brix） | Float | 15.5 |
| 重量 | 单果平均重量（克） | Float | 280.5 |
| 口感 | 口感描述 | String | 肉质细腻，汁多味甜 |
| 适应人群 | 适合食用的人群 | String | 老少皆宜，特别适合血糖稳定人群 |
| 品质小结 | 整体品质评价 | String | 果实饱满，色泽鲜艳，品质优良 |

#### 2.3.3 交互说明
- 质量指标以可视化图表形式展示
- 数值型数据采用进度条或仪表盘展示
- 文字描述采用卡片式布局

## 3. 页面设计规范

### 3.1 整体风格
- 简洁清晰的设计风格
- 自然、清新的农业主题色调
- 信息层级分明，易于阅读

### 3.2 配色方案
| 颜色用途 | 色值 | 说明 |
|---------|------|------|
| 主色调 | #4CAF50 | 自然绿色，代表农业、生态 |
| 辅助色 | #8BC34A | 浅绿色，用于次要元素 |
| 背景色 | #F5F7FA | 浅灰白色，清爽干净 |
| 文字色 | #333333 | 深灰色，主要文字 |
| 次要文字 | #666666 | 中灰色，辅助说明文字 |
| 强调色 | #FF9800 | 橙色，用于重要信息突出 |

### 3.3 布局规范

#### 3.3.1 页面结构
```
+------------------------------------------+
|              页面头部（Logo + 标题）        |
+------------------------------------------+
|                                          |
|  +------------------------------------+  |
|  |         多媒体展示区域（主体）        |  |
|  |      图片轮播 / 视频播放器           |  |
|  +------------------------------------+  |
|                                          |
|  +----------------+  +----------------+  |
|  |   基本信息卡片   |  |   质量信息卡片   |  |
|  +----------------+  +----------------+  |
|                                          |
+------------------------------------------+
```

#### 3.3.2 卡片样式
- 圆角半径：12px
- 阴影：0 4px 12px rgba(0, 0, 0, 0.1)
- 内边距：24px
- 卡片间距：20px
- 背景：白色 #FFFFFF

#### 3.3.3 字体规范
- 标题字体：PingFang SC / Microsoft YaHei
- 正文字体：PingFang SC / Microsoft YaHei
- 字号层级：
  - 页面标题：28px / Bold
  - 模块标题：20px / Medium
  - 正文内容：14px / Regular
  - 辅助说明：12px / Regular

### 3.4 响应式设计
- 桌面端（≥1200px）：三栏布局
- 平板端（768px-1199px）：两栏布局
- 移动端（<768px）：单栏布局

## 4. 页面详细设计

### 4.1 产品溯源页面

#### 4.1.1 页面组成

   - Logo图标

   - 占据页面主体位置（约60%高度）

## 5. 技术实现要点

### 5.1 前端技术要点
- 使用Vue 3 Composition API
- 使用Vite作为构建工具
- 组件化开发，提高代码复用性
- 使用CSS变量管理主题色
- 图片懒加载优化性能

### 5.2 后端技术要点
- Django REST Framework提供API
- SQLite数据库存储
- 支持跨域请求（CORS）
- RESTful API设计规范

### 5.3 数据交互
- 前端通过Axios请求后端API
- 数据格式统一使用JSON
- 错误处理和异常提示
